## 1. 現状サマリー

### サービス概要
- **サービス名**: 楽々省エネ計算
- **URL**: https://rakuraku-energy.archi-prisma.co.jp/
- **対象**: 建築士・設備設計者
- **主要機能**: BEI計算（モデル建物法）、PDF/Excel出力、プロジェクト管理

### 料金の現状
- **実態: 完全無料**（全機能・全ユーザーに開放中）
- 課金システム（Stripe等）は未実装
- ユーザーDBにプラン・課金フィールドなし
- 利用制限なし（計算回数・プロジェクト数に上限なし）
- 「30日間無料トライアル」と記載があるが、期限を制御するコードが存在しない

現在、ページによって異なる料金メッセージが混在しています。

| 箇所 | 現在の表記 | 問題 |

| キャンペーンページ FAQ | 「24時間監視体制で**99.9%の稼働率**」 | Render無料枠では根拠なし |

- 建築士は法的責任を負う専門職であり、サービスの信頼性に敏感
- 表記の矛盾は「このサービス大丈夫か？」という印象を与える
- 「99.9%稼働率」等の根拠のない数値は景品表示法上のリスクあり

## 4. 決めるべきこと

### 論点A: 料金モデルをどうするか

#### 選択肢1: 完全無料を継続
- メリット: ユーザー獲得しやすい、シンプル
- デメリット: 収益ゼロ、持続可能性の疑問
- 適している場合: ポートフォリオ・実績として活用 / 別事業への集客導線

#### 選択肢2: フリーミアム（無料＋有料プラン）
- メリット: 無料で試せる＋収益化の両立
- デメリット: 機能制限の設計・課金システムの開発が必要
- 適している場合: SaaSとして事業化する場合

#### 選択肢3: 有料のみ（無料トライアル付き）
- メリット: 明確な収益モデル
- デメリット: 初期ユーザー獲得のハードル高い
- 適している場合: 十分な機能・信頼性が確立した後

#### 選択肢4: 共同開発パートナーモデル（現状の延長）
- メリット: フィードバックを得ながら開発
- デメリット: 「いつまで無料？」の不透明感
- 適している場合: 開発初期フェーズ

### 論点B: 有料化する場合の価格帯

**競合・参考サービスの価格帯**:
- 省エネ計算の外注: 1件あたり5万円〜
- WEBPRO（国交省）: 無料だが操作が複雑
- 類似SaaS（建築系）: 月額1万円〜5万円が相場

**検討中の価格案（PRICING_STATUS.mdより）**:
- 元々の案: 月額19,800円（現在「調整中」）

### 論点C: 無料/有料で何を分けるか（フリーミアムの場合）

| 機能 | 無料 | 有料（案） |
|------|------|-----------|
| BEI計算 | 月3〜5回 | 無制限 |
| PDF出力 | 「サンプル」透かし | 透かしなし・会社ロゴ入り |
| プロジェクト保存 | 3件まで | 無制限 |
| Excel出力 | なし | あり |
| 計算書の社名カスタマイズ | なし | あり |
| 計算履歴 | 直近5件 | 全件 |
| 優先サポート | なし | あり |

## 5. 即座に修正すべき表記（方針に関わらず）

料金方針が決まる前でも、以下は事実と異なるため修正推奨:

1. **「99.9%の稼働率」** → 根拠がないため削除 or 「稼働状況ページで確認可能」に変更
2. **「有料版と全く同じエンジン」** → 有料版が存在しないため「正規APIによる高精度計算」に変更
3. **「期間終了前にメール通知」** → 通知機能がないため削除
4. **「契約期間の縛りなし / いつでもキャンセル可能」** → 契約がないため削除
