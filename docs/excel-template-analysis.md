## 4. 現在のCELL_MAPPINGとの差分

### 現状 (report.py): 24項目のみ

```
name, climate_zone, building_type, meeting_place_type, total_floor_area,
envelope_system, total_floor, building_height, outer_circumference,
outer_circumference_core, non_ac_core_direction,
uvalue_exterior_wall, uvalue_roof, uvalue_floor,
ac_heat_source_type_cooling, ac_cop_cooling,
ac_heat_source_type_heating, ac_cop_heating,
ac_heatexchanger_efficiency,
v_equipment, l_equipment, l_unit_power,
hw_equipment, hw_efficiency,
ev_present, ev_control_type
```

### 問題点

1. **Named Rangeベースのマッピングだが、テンプレートにはNamed Rangeが入力セル用に定義されていない**
   - テンプレートのNamed Rangesは全てドロップダウンリストの参照先（dataシート）
   - 入力セルにはNamed Rangeが設定されていない
   - → **セル座標ベースの直接書き込みが必要**

2. **様式B1〜I はテーブル形式（複数行）で、現在のkey-value方式では対応不可**
   - 空調熱源は複数台ありうる（例: チラー+ボイラ）
   - 照明器具は室ごとに複数行
   - → **配列データの行展開ロジックが必要**

3. **カバレッジ不足**: 様式B1(開口部), B3(外皮), C2-C4(外気処理/ポンプ/送風機), H(太陽光), I(コージェネ) が完全に未対応

## 5. 新CELL_MAPPING設計方針

### 方針: Named Rangeではなく「シート名+セル座標」で直接書き込む

