## 1. エグゼクティブサマリー

楽々省エネ計算は、建築士向けの**「建材選定 × 公式計算 × メーカー紹介」プラットフォーム**である。

2025年4月の省エネ適合義務化により、年間44.5万棟が新たに省エネ計算の対象となった。
しかし外注はパンク状態（1件5〜15万円）、既存ツールは高額（初年46万円〜）で、
建築士は省エネ計算を**安く・速く・正確に**行う手段を求めている。

楽々省エネ計算は「製品を選ぶだけでBEI適合を確認でき、そのまま審査提出できる」ツールを提供し、
**計算の課金 + メーカーへの紹介手数料**で収益化する。

AI×省エネ計算×メーカー紹介を一気通貫で行うサービスは**市場にまだ存在しない**。

## 2. 市場環境

### 省エネ適合義務化のインパクト

| 項目 | 数値 |
|------|------|
| 新規対象棟数 | 年間 44.5万棟 |
| 外注単価 | 5〜15万円/件 |
| 外注市場の状態 | 需要 > 供給（パンク状態） |
| 既存ツール（SAVE-建築） | 初年46.2万円 + 年6.6万円 |

### 競合状況

| 競合 | 形態 | 弱点 |
|------|------|------|
| SAVE-建築 | パッケージソフト | 高額（初年46万円）、製品推薦なし |
| 省エネ計算代行業者 | 外注サービス | 高い（5〜15万/件）、納期1〜2週間 |
| 自前Excel計算 | 手動 | 時間がかかる、ミスが多い |

**楽々省エネ計算の位置づけ:**
AI製品推薦 + 公式計算 + メーカー紹介まで一気通貫 = **競合ゼロのブルーオーシャン**

## 3. プロダクト

### コアフロー（ユーザー体験）

```
STEP 1  建物情報を選択（用途・地域・面積）
   ↓
STEP 2  製品をカードから選ぶ（AIがおすすめを提示）
        窓 → 断熱 → 空調 → 照明
   ↓
STEP 3  公式APIでBEI計算（国交省エンジン使用）
   ↓
STEP 4  結果表示 + AI改善提案
        「空調をこれに変えるとBEI 0.85 → 0.75」
        → [適用して再計算] ボタンでワンクリック
   ↓
STEP 5  出力
        ・審査提出用 公式PDF
        ・選定製品リスト
        ・[メーカーに見積依頼] → 紹介フロー
```

### 技術構成

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Next.js 14 + Tailwind CSS |
| バックエンド | FastAPI (Python) |
| 計算エンジン | lowenergy.jp 公式API（無料・無認証） |
| AI推薦 | Claude API（Haiku） |
| 決済 | Stripe |
| デプロイ | Docker / Render |

## 4. ビジネスモデル: 価格.com型（建築士無料 + メーカー課金）

### 建築士は完全無料

BEI計算、AI製品推薦、公式PDF出力、メーカー見積依頼 — 全て無料。
「無料」が最強の参入障壁になる（後発は「無料 + 品質上位」でないと勝てない）。

### 収益源は3つ

```
┌─────────────────────────────────────────┐
│  収益源 ①: メーカースポンサー契約        │
│                                          │
│  ・パートナー掲載料  30〜100万円/月/社  │
│  ・AI推薦での優先表示                    │
│  ・製品選択データ・市場レポート提供       │
└─────────────────────────────────────────┘
           ＋
┌─────────────────────────────────────────┐
│  収益源 ②: リード課金                    │
│                                          │
│  ・見積依頼1件あたり ¥3,000〜5,000      │
│  （価格.comのクリック課金に相当）         │
└─────────────────────────────────────────┘
           ＋
┌─────────────────────────────────────────┐
│  収益源 ③: 紹介成約手数料               │
│                                          │
│  建築士が製品を選定                       │
│    → メーカーに見積依頼ボタン            │
│    → 成約時 2〜5% の紹介手数料          │
│                                          │
│  1案件あたり 4.6万〜29.5万円            │
└─────────────────────────────────────────┘
```

### なぜ建築士無料か（価格.comの教訓）

| 比較 | 価格.com | 楽々省エネ計算 |
|------|---------|--------------|
| ユーザー | 消費者（無料） | 建築士（無料） |
| 課金対象 | ショップ（クリック課金） | メーカー（スポンサー+リード課金） |
| 営業利益率 | 47% | 目標 90%+ |
| ネットワーク効果 | ユーザー↑→ショップ↑→価格↓→ユーザー↑ | 建築士↑→メーカー参加↑→製品充実→建築士↑ |

### 原価構造（1計算あたり）

| 項目 | コスト |
|------|--------|
| lowenergy.jp API | 0円 |
| Claude API（製品推薦 × 3回） | 150〜300円 |

## 7. 競合優位性（参入障壁）

| 障壁 | なぜ真似できないか |
|------|-------------------|
| **メーカーとの人脈** | YKK AP会長、パナソニックテクノとの直接関係。営業では築けない |
| **YKKグループ統合** | 2026年4月にYKK AP + パナソニック住宅事業が同一グループ化。窓+設備を一気通貫でカバーするプラットフォームはグループ戦略と完全に合致 |
| **製品DB** | パートナーからの優先データ提供。公開+非公開データの組み合わせ |
| **AI推薦の精度** | 計算実績が溜まるほど「この条件ならこの製品」の精度が向上 |
| **ネットワーク効果** | 建築士が増える → メーカーが参加 → 製品が充実 → さらに建築士が増える |
| **無料という参入障壁** | 最初から無料なら、後発は「無料 + 品質上位」が必要。極めて困難 |
| **先行者優位** | 義務化直後の需要爆発期。このタイミングで市場を取れば後発は追いつけない |

## 9. リスクと対策

| リスク | 対策 |
|--------|------|
| 公式APIのレート制限 | リトライ+バックオフで安定化。利用量に応じた負荷分散 |
| メーカーが紹介手数料に消極的 | ツール単体でも月額収益で黒字（粗利95%） |
| 競合が同様機能を追加 | パートナーシップとAI精度で差別化。先行者優位を活かす |
| 計算結果の正確性への疑義 | 公式API（国交省エンジン）のみ使用。結果はそのまま審査提出可能 |
