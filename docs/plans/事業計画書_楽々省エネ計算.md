## 1. エグゼクティブサマリー

楽々省エネ計算は、建築士向けの**「建材選定 × 公式計算 × メーカー紹介」プラットフォーム**である。

2025年4月の省エネ適合義務化により、年間44.5万棟が新たに省エネ計算の対象となった。
しかし外注はパンク状態（1件5〜15万円）、既存ツールは高額（初年46万円〜）で、
建築士は省エネ計算を**安く・速く・正確に**行う手段を求めている。

楽々省エネ計算は「製品を選ぶだけでBEI適合を確認でき、そのまま審査提出できる」ツールを提供し、
**計算の課金 + メーカーへの紹介手数料**で収益化する。

AI×省エネ計算×メーカー紹介を一気通貫で行うサービスは**市場にまだ存在しない**。

| 競合 | 形態 | 弱点 |
|------|------|------|

**楽々省エネ計算の位置づけ:**
AI製品推薦 + 公式計算 + メーカー紹介まで一気通貫 = **競合ゼロのブルーオーシャン**

## 3. プロダクト

### コアフロー（ユーザー体験）

```
STEP 1  建物情報を選択（用途・地域・面積）
   ↓
STEP 2  製品をカードから選ぶ（AIがおすすめを提示）
        窓 → 断熱 → 空調 → 照明
   ↓
STEP 3  公式APIでBEI計算（国交省エンジン使用）
   ↓
STEP 4  結果表示 + AI改善提案
        「空調をこれに変えるとBEI 0.85 → 0.75」
        → [適用して再計算] ボタンでワンクリック
   ↓
STEP 5  出力
        ・審査提出用 公式PDF
        ・選定製品リスト
        ・[メーカーに見積依頼] → 紹介フロー
```

### 技術構成

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Next.js 14 + Tailwind CSS |
| バックエンド | FastAPI (Python) |
| 計算エンジン | lowenergy.jp 公式API（無料・無認証） |
| AI推薦 | Claude API（Haiku） |
| 決済 | Stripe |
| デプロイ | Docker / Render |

## 7. 競合優位性（参入障壁）

| 障壁 | なぜ真似できないか |
|------|-------------------|
| **メーカーとの人脈** | YKK AP会長、パナソニックテクノとの直接関係。営業では築けない |
| **製品DB** | パートナーからの優先データ提供。公開+非公開データの組み合わせ |
| **AI推薦の精度** | 計算実績が溜まるほど「この条件ならこの製品」の精度が向上 |
| **ネットワーク効果** | 建築士が増える → メーカーが参加 → 製品が充実 → さらに建築士が増える |
| **先行者優位** | 義務化直後の需要爆発期。このタイミングで市場を取れば後発は追いつけない |

## 9. リスクと対策

| リスク | 対策 |
|--------|------|
| 公式APIのレート制限 | リトライ+バックオフで安定化。利用量に応じた負荷分散 |
| メーカーが紹介手数料に消極的 | ツール単体でも月額収益で黒字（粗利95%） |
| 競合が同様機能を追加 | パートナーシップとAI精度で差別化。先行者優位を活かす |
| 計算結果の正確性への疑義 | 公式API（国交省エンジン）のみ使用。結果はそのまま審査提出可能 |
